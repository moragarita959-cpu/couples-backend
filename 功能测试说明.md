# 功能测试说明

## 测试环境

### 前置条件
1. 已搭建好uniCloud环境，创建了相应的云函数和数据库
2. 已配置好manifest.json中的云开发配置
3. 已安装必要的依赖
4. 拥有两个测试手机号用于测试情侣绑定功能

### 测试设备
- 微信开发者工具
- 移动设备（iOS/Android）

## 测试功能列表

### 1. 用户认证与情侣绑定功能

#### 1.1 手机号验证码登录
**测试步骤：**
1. 打开应用，进入登录页面
2. 输入测试手机号
3. 点击"获取验证码"
4. 输入收到的验证码
5. 点击"登录"

**预期结果：**
- 登录成功，跳转到首页
- 用户信息正确存储

#### 1.2 情侣绑定请求的发送和接收
**测试步骤：**
1. 用户A登录应用
2. 进入绑定页面，输入用户B的手机号
3. 点击"发送绑定请求"
4. 用户B登录应用
5. 查看通知列表

**预期结果：**
- 用户A成功发送绑定请求
- 用户B收到绑定请求通知

#### 1.3 绑定关系的确认
**测试步骤：**
1. 用户B在通知中点击绑定请求
2. 点击"确认绑定"

**预期结果：**
- 绑定成功
- 用户A收到绑定成功通知
- 双方进入首页后显示情侣关系信息

#### 1.4 绑定状态的实时同步
**测试步骤：**
1. 用户A和用户B同时在线
2. 用户A发送绑定请求
3. 用户B确认绑定

**预期结果：**
- 用户A的绑定状态实时更新为已绑定
- 用户B的绑定状态实时更新为已绑定

### 2. 实时聊天功能

#### 2.1 消息发送和接收
**测试步骤：**
1. 用户A和用户B已绑定为情侣
2. 用户A进入聊天页面，输入消息并发送
3. 用户B进入聊天页面

**预期结果：**
- 用户A的消息发送成功
- 用户B实时收到消息
- 消息按时间顺序显示

#### 2.2 消息已读状态
**测试步骤：**
1. 用户A发送消息给用户B
2. 用户B查看消息

**预期结果：**
- 用户B的消息显示为已读
- 用户A看到消息状态变为已读

#### 2.3 实时消息推送
**测试步骤：**
1. 用户B后台运行应用
2. 用户A发送消息给用户B

**预期结果：**
- 用户B收到消息推送通知
- 点击通知进入聊天页面，显示新消息

- 日记信息正确关联到情侣关系

#### 4.1 连续聊天天数统计

2. 进入首页查看统计数据

- 连续聊天天数正确显示

- 数据实时更新

## 测试注意事项

1. **数据安全**：测试过程中避免使用真实的个人信息
2. **网络环境**：确保测试设备网络连接稳定
3. **权限设置**：确保应用有必要的权限（如相机、相册访问权限）
4. **错误处理**：测试各种错误场景，如网络中断、输入错误等
5. **性能测试**：测试应用在大量数据下的性能表现

## 故障排查

### 常见问题及解决方案

1. **云函数调用失败**
   - 检查云函数是否已部署
   - 检查云函数权限设置
   - 检查网络连接

2. **消息推送不及时**
   - 检查uniCloud实时监听设置
   - 检查设备网络状态
   - 检查应用后台运行权限

3. **数据存储失败**
   - 检查数据库权限设置
   - 检查数据格式是否正确
   - 检查云函数错误日志

4. **统计数据不准确**
   - 检查统计逻辑是否正确
   - 检查数据同步是否及时
   - 检查数据库查询条件是否正确

## 测试完成标准

1. 所有功能模块测试通过
2. 无严重错误或异常
3. 应用性能符合预期
4. 用户体验流畅

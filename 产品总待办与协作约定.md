# 产品总待办与协作约定（持续更新）

> 目的：把所有“未实现的想法 + 待办 + 协作要求”先统一记录，后续新增内容也先写入本文件，再进入开发。

## 1. 使用规则（必须遵守）

1. 任何新想法、新需求、新待办，先登记到本文档，再排期实现。
2. 涉及实现细节、交互细节、边界条件时，必须及时与你核对后再继续。
3. 当任务需要额外 skill 时，允许由助手自行下载并接入可用 skill（在合规前提下）。
4. 每次迭代后，更新本文档中的状态、优先级和备注，避免口头需求丢失。

## 2. 当前阶段目标（总原则）

1. 先把功能框架补齐，再做深度优化。
2. 现有已上线/已完成功能，后续都要进入“全面优化”阶段（体验、性能、稳定性、UI 一并优化）。

## 5. 协作与确认机制（执行要求）

1. 需求确认：
   - 在开始做每个功能前，先确认“目标、范围、验收标准”。
2. 细节确认：
   - 遇到文案、交互、数据结构、边界行为等细节时，第一时间与你核对。
3. 变更确认：
   - 若实现中发现方案要调整，先说明影响，再等待你确认。
4. 记录留痕：
   - 每次确认结论必须回写到本文档（或关联设计文档）以便追踪。

## 6. 全量优化计划（针对已有功能）

后续对“已有功能”统一做以下优化：

1. UI/UX：视觉一致性、层级、可读性、动效与反馈。
2. 性能：首屏速度、卡顿点、资源占用。
3. 稳定性：异常兜底、错误提示、边界处理。
4. 可维护性：代码结构、模块职责、文档同步。
